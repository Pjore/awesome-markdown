# awesome-markdown

Lightweight, git-backed kanban system. Pure client-side UI, pluggable persistence
providers, and an independent sync-engine for git and external-change notifications.

## Architecture

Monorepo with pnpm workspaces:

- `packages/contracts` — Zod v4 schemas + inferred TypeScript types (shared across all packages)
- `packages/provider-localstorage` — In-browser localStorage provider (no server required)
- `apps/kanban-ui` — React 19 + Vite 8 + Tailwind v4 SPA with @dnd-kit drag-and-drop
- `apps/provider-fs` — Fastify v5 sidecar: persists content as markdown files with YAML frontmatter
- `apps/sync-engine` — Watches `content/`, auto-commits, push/pull via git, emits SSE events

## Tech Stack

| Layer | Tech |
|-------|------|
| UI | React 19, Vite 8 (Rolldown + Oxc), Tailwind v4, @dnd-kit |
| API/sidecar | Fastify v5 + `fastify-type-provider-zod` |
| Validation | Zod v4 |
| Git | simple-git + GitHub Fine-Grained PAT |
| File watching | chokidar |
| Markdown | gray-matter |
| Live channel | Native SSE |
| Monorepo | pnpm workspaces |

## Dev Commands

```bash
# Install dependencies
pnpm install

# Type-check and lint (quality gates)
pnpm typecheck && pnpm lint
```

See `apps/*/README.md` for per-app dev commands and port mappings.

## File Constraints

| Type | Limit |
|------|-------|
| TypeScript source files | 400 lines max |
| AI files (instructions, prompts, skills) | 600 words max |
